`abcstat` XeLaTeX project. The work spanned two sessions (continued from a prior session)

## 2. Fatal Error: `! Missing $ inserted`

### 2.1 Symptom

```
! Missing $ inserted.
l.34       \subsubitem i}
                         }({\symsfup {f}})$, \hyperpage{363}
```

XeLaTeX crashed when reading `xsym.ind` (the symbol index generated by makeindex).

### 2.2 Root Cause

The commands `\epi`, `\hyp`, and `\linspan` in `common/sty/math.sty` were defined using
`\newcommand` with `\!` (negative thin space) inside `\mathsfup{...}`:

```latex
\newcommand{\epi}    [0]{{\mathsfup{e\!p\!i}}}
\newcommand{\hyp}    [0]{{\mathsfup{h\!y\!p}}}
\newcommand{\linspan}[0]{{\mathsfup{s\!p\!a\!n}}}
```

These commands were indexed via `\hxs{...}` (defined as `\newcommand{\hxs}[1]{#1\index[xsym]{$#1$}}`).
Under TeX Live 2025 / unicode-math v0.8r, `\mathsfup` expands its argument during `\write`
to the `.idx` file. This caused the `\!` kern to lose its backslash, producing a bare `!`
in the `.idx` file. Makeindex treats bare `!` as a level separator, generating a malformed
multi-level index entry. XeLaTeX then read the malformed `xsym.ind` and encountered
`\symsfup` outside math mode.

### 2.3 Fix

Changed `\newcommand` to `\DeclareRobustCommand` for all three commands
(`common/sty/math.sty`, lines 1313–1315):

```latex
\DeclareRobustCommand{\epi}    {{\mathsfup{e\!p\!i}}}
\DeclareRobustCommand{\hyp}    {{\mathsfup{h\!y\!p}}}
\DeclareRobustCommand{\linspan}{{\mathsfup{s\!p\!a\!n}}}
```

`\DeclareRobustCommand` writes the command *name* (not its expansion) to `.idx` files,
so `\!` kerns are preserved in typeset output while the index entry remains clean.

**Non-fixes tried first:**
- `\protect\!` — did not prevent expansion during `\write`
- Removing `\!` entirely — fixed the error but eliminated the kerning

## 7. amsmath `\atopwithdelims` Warning

`Package amsmath Warning: Foreign command \atopwithdelims`

### 7.1 Root Cause

`\bcoef` in `common/sty/math.sty` was defined using the plain-TeX `\choose` primitive:

```latex
\newcommand{\bcoef}[2]{{#1\choose#2}}
```

`\choose` expands internally to `\atopwithdelims()`, which amsmath flags as a foreign
(non-amsmath) delimiter command.

### 7.2 Fix

Changed to amsmath's proper command:

```latex
\newcommand{\bcoef}[2]{\binom{#1}{#2}}
\newcommand{\bcoefnk}[0]{\binom{n}{k}}
```

## 9. hyperref Warnings

### 9.1 Token Not Allowed in PDF String

`Package hyperref Warning: Token not allowed in a PDF string (Unicode): removing \fntgnuFreeSerif`

A font-switching command appeared in a section title in `common/sums.tex`, which hyperref
tried to write into the PDF bookmark string. Fix: wrap with `\texorpdfstring`:

```latex
% Before:
\subsection{Weighted {\fntFreeSerif\textphi}-means}

% After:
\subsection{Weighted \texorpdfstring{{\fntFreeSerif\textphi}}{phi}-means}
```

### 9.2 Bookmark Anchor Conflict

`Package hyperref Warning: The anchor of a bookmark and its parent's must not be the same.`

In `common/backmat.tex`, `\addcontentsline` appeared without `\phantomsection` immediately
before `\bibliography`. Fix:

```latex
% Before:
\addcontentsline{toc}{section}{References}
\bibliography{...}

% After:
\phantomsection\addcontentsline{toc}{section}{References}
\bibliography{...}
```

## 10. Corrupted Characters in `laplace.tex`

`Missing character: There is no (U+FFFD) in font`

Lines 9–11 of `common/laplace.tex` contained a French language quotation with two
U+FFFD (Unicode replacement character) bytes and several malformed LaTeX accent commands
(`/'e`, `/<e`). These were corrupted during a prior file encoding conversion.

**Fixed text:**

```latex
% Before (with corruption):
{La langue de l[U+FFFD]analyse, la plus parfaite de toutes les langues,
[U+FFFD]tant par elle-m/<eme un puissant instrument de d/'ecouvertes;
ses notations, lorsqu'elles sont n/'ecessaires et heureusement imagin/'ees,

% After:
{La langue de l'analyse, la plus parfaite de toutes les langues,
\'etant par elle-m\^eme un puissant instrument de d\'ecouvertes;
ses notations, lorsqu'elles sont n\'ecessaires et heureusement imagin\'ees,
```

The repairs were done using Python binary file I/O to directly replace the corrupted bytes.

## 11. stmaryrd Font Size Warnings

`LaTeX Font Warning: Font shape 'U/stmry/m/n' in size <3.58> not available`

### 11.1 Root Cause

The system `Ustmry.fd` (stmaryrd font descriptor) only declares sizes ≥5pt and uses
`gen * stmary` for integer sizes 5–10. Fractional sizes (e.g., 3.58pt, 6.53pt) appearing
in deeply nested subscripts or footnotes had no matching declaration.

### 11.2 Fix

Created a local `abcstat/Ustmry.fd` that XeLaTeX finds before the system copy (the build
directory is always checked first). Uses explicit size ranges to handle all fractional
requests without substitution warnings:

```latex
\DeclareFontFamily{U}{stmry}{}
\DeclareFontShape{U}{stmry}{m}{n}
   {  <-5>   stmary5
      <5-6>  stmary5
      <6-7>  stmary6
      <7-8>  stmary7
      <8-9>  stmary8
      <9-10> stmary9
      <10->  stmary10
   }{}
\endinput
```

**Non-fix tried first:** Adding `\DeclareFontShape` in `fonts.sty` after
`\usepackage{stmaryrd}`. This caused `! LaTeX Error: Font family 'U+stmry' unknown`
because `.fd` files are loaded lazily (only when the font is first used, not at package
load time), so the font family was not yet declared when the preamble code ran.

## 14. Key Technical Insights

1. **`\DeclareRobustCommand` vs `\newcommand` for indexed commands:**
   Commands used inside `\index{}` that contain fragile content (e.g., `\!`) must be
   declared robust. `\DeclareRobustCommand` writes the command name to `.idx`, not its
   expansion.

2. **LaTeX `.fd` files load lazily:**
   Font definition (`.fd`) files are loaded on first font use, not at package load time.
   `\DeclareFontShape` in the preamble fails with "Font family unknown" if the family
   hasn't been used yet. The correct override mechanism is a local `.fd` file in the
   build directory, which is found by the TeX file-search system before the system copy.

3. **`sed -i` and CRLF files on Windows:**
   `sed -i` in Git Bash on Windows does not reliably process files with CRLF line endings.
   Use Python with binary file I/O (`open(path, 'rb')`/`'wb'`) for reliable byte-level
   editing.

4. **Geometry file identification:**
   When a LaTeX document loads multiple geometry-related `.sty` files, check `main.tex`
   carefully to identify which file is *actually* loaded. Editing the wrong file has no
   effect.

5. **`\ProvidesPackage` in project-local `.sty` files:**
   When packages are loaded via relative paths (`\usepackage{../common/sty/name}`), the
   declared package name in `\ProvidesPackage{name}` must match exactly or be omitted.
   Omitting `\ProvidesPackage` (consistent with the rest of the project) is the cleanest
   approach.

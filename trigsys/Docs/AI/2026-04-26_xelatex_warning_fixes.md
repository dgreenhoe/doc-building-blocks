## 2. Fatal Error: `! Missing $ inserted` (xsym.ind)

### 2.1 Symptom

```
[12] (./xsym.ind
! Missing $ inserted.
l.23     \subitem
                  \
! Emergency stop.
```

XeLaTeX crashed when reading `xsym.ind` (the symbol index generated by makeindex).

### 2.2 Root Cause

`\spPW` in `common/sty/math.sty` was defined with `\!\!` (double negative thin space)
between `\mbfitsansP` and `\mbfitsansW`:

```latex
\newcommand{\spPW}[0]{\mbfitsansP\!\!\mbfitsansW}  % Paley-Wiener class of functions
```

When `\spPW` was used inside `\hxs{\spPW_\sigma^2}` in `common/transop.tex`, the
`\index[xsym]{$\spPW_\sigma^2$}` call wrote the expansion to `xsym.idx`. Under
TeX Live 2025 / unicode-math, `\!\!` expanded to bare `!!` in the `.idx` file.
Makeindex treats bare `!` as a level separator, so `!!` created a malformed three-level
index entry:

```
\item $\mbfitsansP \
  \subitem \
    \subsubitem \mbfitsansW _\mitsigma ^2$
```

XeLaTeX then encountered `\subitem \` (bare backslash outside math) and crashed.

### 2.3 Fix

Changed `\newcommand` to `\DeclareRobustCommand` (`common/sty/math.sty`, line 1483):

```latex
\DeclareRobustCommand{\spPW}{\mbfitsansP\!\!\mbfitsansW}  % Paley-Wiener class of functions
```

`\DeclareRobustCommand` writes the command *name* to `.idx`, not its expansion,
so `\!\!` never reaches makeindex.

**Mechanism:** This is the same root cause as the `\epi`/`\hyp`/`\linspan` crash fixed
in `abcstat` (2026-03-04); those three commands were already `\DeclareRobustCommand` in
`math.sty` from that prior fix. `\spPW` was a new occurrence.

## 3. fancyhdr Geometry Warnings

### 3.1 Symptom

```
Package fancyhdr Warning: \headheight is too small (14.79541pt):
(fancyhdr)                Make it at least 17.99998pt.
Package fancyhdr Warning: \footskip is too small (19.91692pt):
(fancyhdr)                Make it at least 20.10808pt.
```

### 3.2 Root Cause and Fix

The active geometry file for `trigsys` is `common/sty/report.sty` (not `xsd.sty`, which
is loaded only by `abcstat`). It had:

```latex
headheight=5.2mm,headsep=5mm,footskip=7mm,
```

5.2 mm ≈ 14.74 pt (below the required 17.99998 pt);  7 mm ≈ 19.84 pt (below 20.10808 pt).

Fixed to (`common/sty/report.sty`, line 50):

```latex
headheight=30pt,headsep=5mm,footskip=21pt,
```

The extra headroom (30 pt vs the minimum 18 pt) matches the value used in `xsd.sty`
for the `abcstat` project, which has the same table-based header.

**Cascade effect:** Fixing `headheight` also eliminated the three
`ignored error: Infinite glue shrinkage found in box being split` messages that had
appeared on pages 8–9 in the earlier broken build log (those were fancyhdr's internal
`\vss` from trying to place an oversized header into an undersized box). Note that three
"Infinite glue shrinkage" messages remain on pages 9–11 of the working build; see §8.

## 4. hyperref Bookmark Anchor Warning (front.tex)

### 4.1 Symptom

```
Package hyperref Warning: The anchor of a bookmark and its parent's must not
(hyperref)                be the same. Added a new anchor on input line 19.
```

### 4.2 Root Cause and Fix

Two consecutive `\addcontentsline` calls in `trigsys/front.tex` shared the same anchor:

```latex
\addcontentsline{toc}{chapter}{Front matter}
\addcontentsline{toc}{section}{Front cover}       % ← line 19: same anchor as parent
```

Fix: add `\phantomsection` before the section entry to force a new anchor:

```latex
\addcontentsline{toc}{chapter}{Front matter}
\phantomsection\addcontentsline{toc}{section}{Front cover}
```

## 5. hyperref Bookmark Anchor Warning (backmatr.tex)

Preventive fix applied to `common/backmatr.tex` to avoid the same warning once the build
could reach the back matter:

```latex
% Before:
\addcontentsline{toc}{section}{References}
\bibliography{...}

% After:
\phantomsection\addcontentsline{toc}{section}{References}
\bibliography{...}
```

## 6. amsmath `\atopwithdelims` Warning

### 6.1 Symptom

```
Package amsmath Warning: Foreign command \atopwithdelims;
(amsmath)                \frac or \genfrac should be used instead
(amsmath)                 on input line 883.
```

### 6.2 Root Cause

Multiple files used the plain-TeX `\choose` primitive inside `{...}` groups. amsmath
flags `\choose` (which expands to `\atopwithdelims()`) as a "foreign command."

### 6.3 Files Modified and Counts

| File | Replacements |
|------|-------------|
| `common/harPoly.tex` | 199 |
| `common/harTrig.tex` | 1 |
| `common/calculus.tex` | 2 |
| `common/polynom.tex` | 5 |

### 6.4 Fix

All occurrences of `{A\choose B}` replaced with `\binom{A}{B}` using a Python
binary-file replacement script (required for CRLF safety on Windows).

Simple cases handled by regex `\{([^{}]*?)\\choose\s*([^{}]*?)\}` →
`\binom{EXPR1}{EXPR2}` (with whitespace trimming).

Three cases in `harPoly.tex` had `\frac{n}{2}` as the second argument (nested braces,
not matched by the simple regex): `{n\choose\frac{n}{2}}` → `\binom{n}{\frac{n}{2}}`,
edited manually.

## 7. Undefined Citation Warnings (Resolved Automatically)

The original log showed 10 undefined citation warnings (paine2000, housman1936,
hardy1940, ewen1961, ewen1950, heijenoort, Bendiner1980, descartes_rules,
descartes_rules_eng, cajori2). All citations are present in
`common/bib/mathmisc.bib` and `common/bib/mathhist.bib` and were present in `mainx.bbl`.

**Cause:** The `xsym.ind` fatal error (§2) killed XeLaTeX at page 12, before it could
reach `\bibliography` in `common/backmatr.tex` (near the end of the document). On every
pass, `\bibliography` was never executed, so natbib could not resolve any citations.

**Resolution:** These warnings vanished automatically once the `\spPW` fatal error was
fixed and the build could complete past page 12.

## 8. xdvipdfmx Transparency Warnings

### 8.1 Symptom

```
%%%% WARNING: Transparency operations ignored - need to use -dALLOWPSTRANSPARENCY
```

Appeared 20 times in the xdvipdfmx output, once per PSTricks graphic that uses
transparency.

### 8.2 Root Cause

Ghostscript 10.x (shipped with TeX Live 2025) requires the explicit flag
`-dALLOWPSTRANSPARENCY` to process PS transparency operations. Without it,
all transparency is silently discarded and the warning is emitted. The flag
must be added to the `D` template (the Ghostscript invocation template) in
`dvipdfmx.cfg`, not to the xdvipdfmx command line.

### 8.3 Fix

Created the user-override config at
`TEXMFCONFIG/dvipdfmx/dvipdfmx.cfg`
(`C:/Users/Daniel Greenhoe/.texlive2025/texmf-config/dvipdfmx/dvipdfmx.cfg`),
a full copy of the system `texmf-config/dvipdfmx/dvipdfmx.cfg` with
`-dALLOWPSTRANSPARENCY` inserted after `-dEPSCrop` in the `D` line:

```
D  "rungs -q -dSAFER -dNOPAUSE -dBATCH -dEPSCrop -dALLOWPSTRANSPARENCY -sPAPERSIZE=a0 ..."
```

**Why full copy instead of partial override:** xdvipdfmx reads exactly one
`dvipdfmx.cfg` (the first one found in the kpse search path). If the user
file contains only the `D` line, all font map settings (`f pdftex.map`,
`f kanjix.map`, `f ckx.map`), the PDF version (`V 7`), and other settings
from the system config are lost. The user file must be a complete replacement.

**How to find the active config file:**

```
kpsewhich -progname=dvipdfmx -format=othertext dvipdfmx.cfg
```

**Why not edit the system config directly:**
`C:/p/texlive/2025/texmf-config/dvipdfmx/dvipdfmx.cfg` is write-protected
(EPERM in Git Bash). The user TEXMFCONFIG location
(`C:/Users/Daniel Greenhoe/.texlive2025/texmf-config/`) takes precedence
in the kpse search order.

### 8.1 Dingbat Font Space Probe

```

```

Occurs three times during fontspec font family initialization. fontspec probes the font

### 8.2 Infinite Glue Shrinkage (multicol + theindex)

```
ignored error: Infinite glue shrinkage found in box being split [9
ignored error: Infinite glue shrinkage found in box being split
ignored error: Infinite glue shrinkage found in box being split
```

Occurs on pages 9–11 (the symbol table and symbol index pages in the front matter).

**Root cause:** `trigsys/front.tex` wraps `\input{xsym.ind}` inside
`\begin{multicols}{4}...\end{multicols}`. The `xsym.ind` file begins with
`\begin{theindex}`, which is a two-column environment. The `multicol` package uses
`\vsplit` to split content across columns; `theindex` contains glue with infinite
shrinkability, triggering the warning.

**Status:** TeX marks these as "ignored errors" and continues; the PDF output is
unaffected. Resolving this would require restructuring the symbol index to avoid
nesting `theindex` inside `multicol`.

## 10. Key Technical Insights

1. **`\DeclareRobustCommand` vs `\newcommand` for indexed commands:**
   Any command used inside `\index{}` that contains fragile content (including `\!`
   negative thin spaces) must be `\DeclareRobustCommand`. Robust commands write their
   *name* to `.idx`; fragile commands write their expansion, which can corrupt the index
   when special characters like `!` appear in the expansion.

2. **Active geometry file identification:**
   `trigsys` loads `common/sty/report.sty` (not `xsd.sty`). The geometry, header, and
   footer for `trigsys` are configured in `report.sty`. Editing `xsd.sty` for `trigsys`
   geometry changes has no effect.

3. **`{A\choose B}` and nested braces:**
   A simple `[^{}]*?` regex does not match arguments containing nested braces
   (e.g., `{n\choose\frac{n}{2}}`). Such cases require manual substitution or a
   brace-aware parser.

4. **`sed -i` and CRLF files on Windows:**
   `sed -i` does not reliably edit CRLF files in Git Bash on Windows. Python with binary
   file I/O (`open(path, 'rb')`/`'wb'`) was used for the `\choose`→`\binom` mass
   substitution.

5. **Citation warnings cascade from fatal error:**
   The `\bibliography` command in `backmatr.tex` is near the end of the document.
   Any fatal error that stops XeLaTeX before reaching the back matter prevents citation
   resolution on every pass. Fixing the fatal error is sufficient to resolve all citation
   warnings.

6. **xdvipdfmx `-D` flag vs. Ghostscript flags in `dvipdfmx.cfg`:**
   `-dALLOWPSTRANSPARENCY` is a Ghostscript flag, not an xdvipdfmx flag. It must be
   added inside the `D` template string in `dvipdfmx.cfg`, not passed directly on the
   xdvipdfmx command line. Adding it to `OPT_XDV2PDF` in the makefile causes a fatal
   xdvipdfmx error ("Are you sure this is a DVI file?").

7. **TEXMFCONFIG override takes full precedence:**
   xdvipdfmx reads exactly one `dvipdfmx.cfg` — the first found by kpse in the search
   order (TEXMFCONFIG > TEXMFHOME > texmf-local > texmf-dist). A user-level override
   must be a *complete* replacement of the system config, not just the changed line, or
   all font map settings and PDF version settings from the system config are lost.
   Use `kpsewhich -progname=dvipdfmx -format=othertext dvipdfmx.cfg` to confirm which
   file is active.

**Future direction recommendations:**
- The symbol index section in `front.tex` nests `\begin{theindex}` inside
  `\begin{multicols}{4}`. Consider restructuring to use imakeidx's native multicolumn
  support (`columns=4` option to `\makeindex`) instead, which would eliminate the
  `multicol`/`theindex` interaction causing the "Infinite glue shrinkage" warnings.
- The `\spPW` command uses `\!\!\` for visual kerning in typeset output. Consider
  whether the kerning is still needed or if `\mbfitsansP\mbfitsansW` (without kern)
  is visually acceptable — it would eliminate the need for `\DeclareRobustCommand`.
- Investigate whether other commands in `math.sty` with `\!` inside their bodies are
  also used in indexed contexts; apply `\DeclareRobustCommand` proactively to any
  such commands.
- The `~/.texlive2025/texmf-config/dvipdfmx/dvipdfmx.cfg` user override will need
  to be re-created after each TeX Live major version upgrade (the path encodes `2025`).
  Consider adding this file to version control or a setup script.
